**Goal:** Prevent server bottlenecks, rate-limits, and crashes when 250 users trigger events simultaneously.

* **Action:** Locate the `setInterval` block that loops through the leaderboard and runs `User.findOneAndUpdate`.

### 2. Server Crash Recovery & Persistence
* **File:** `realtime/game/GameEngine.ts`, `realtime/server.ts`

* **Files:** `realtime/game/GameEngine.ts`, `stores/useGameSyncStore.ts`, `components/game/GameHUD.tsx`

## 🎨 PHASE 3: FRONTEND UX & ANTI-CHEAT
**Goal:** Guarantee a seamless player experience on mobile devices and secure the integrity of the game.

### 1. Image Preloading & CDN
* **Action:** Ensure all images are routed through a CDN. When the server prepares the *next* question in the background, send a silent socket event `preload_asset` containing the image URL.
* **Client-Side:** Render a hidden `<link rel="preload" as="image" href={url} />` so the image is already cached in the browser by the time the next question starts.

### 2. Robust Reconnection Handling
* **Files:** `stores/useGameSyncStore.ts`, `realtime/sockets/game.ts`
* **Action:** Mobile browsers pause WebSockets when the tab is backgrounded. In the `socket.on('connect')` listener within Zustand, emit an explicit `request_full_sync` event.
* **Server-Side:** Respond to `request_full_sync` by immediately returning `getStateForClient(usn)` to ensure the user is perfectly synced upon waking the device.

### 3. Submission Debouncing & UI Lock
* **Files:** `components/game/QuestionPhase.tsx`
* **Action:** The moment a user clicks "Submit", immediately disable the button, set opacity to 0.5, and change the text to "Submitting...". Keep it disabled until the `answer_result` is received or a 3-second timeout clears it.

### 4. Screen Wake Lock API

* **Action:** Implement `navigator.wakeLock.request('screen')` whenever `phase === 'question_active'`. Listen for visibility changes to re-request the lock if the user switches tabs and comes back.

### 5. Strict Sanitization & Payload Security
* **Files:** `realtime/game/GameEngine.ts`
* **Action 1 (Sanitization):** Update the `handleAnswer` logic to comprehensively strip punctuation, remove all whitespace (`.replace(/\s+/g, '')`), and apply `.toUpperCase()` before comparing the user's answer to the correct answer.
